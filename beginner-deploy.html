Lesson 05・正式上線

網站發布教學

把本機網站發布到 GitHub Pages，取得公開網址，讓任何人都能用瀏覽器查看你的作品。

1. 發布前先確認

  • 所有 HTML、CSS、JavaScript 與圖片都在專案資料夾內。
  • 檔名不要使用空格，大小寫必須與連結完全相同。
  • 連結使用相對路徑，例如 styles.css，不要使用電腦上的完整路徑。
  • 先在本機逐頁點擊導覽，確認沒有遺漏頁面。

2. 將首頁命名為 index.html

多數網站服務會自動尋找 index.html 作為首頁。你的網站目前以 index.html 為首頁，發布前可將它改名為 index.html，並把其他頁面中連回首頁的網址同步更新。

<a href="index.html">課程首頁</a>

改名後要再次測試所有「課程首頁」連結，並建立一個新的 Git 提交。

3. 開啟 GitHub Pages

  1. 進入 GitHub 上的網站儲存庫。
  2. 選擇 Settings，再選擇左側的 Pages。
  3. 在 Build and deployment 的 Source 選擇 Deploy from a branch。
  4. Branch 選擇 main，資料夾選擇 /(root)，按下 Save。
  5. 等待幾分鐘後重新整理頁面，GitHub 會顯示網站網址。

網址通常會是：

https://你的帳號.github.io/儲存庫名稱/

4. 更新已發布的網站

發布之後不必重新設定 Pages。修改檔案並推送到 main 分支，GitHub 就會自動重新發布。

git add .
git commit -m "更新網站內容"
git push

更新通常需要一點時間。若畫面仍是舊版，可稍候後重新整理，或使用無痕視窗確認。

5. 常見問題

網站出現 404

確認首頁是否叫做 index.html，以及 Pages 是否選到正確的 main 分支。

CSS 或圖片沒有顯示

檢查路徑大小寫，不要用 C:\... 這類只存在自己電腦上的路徑，也不要在相對路徑最前面多加斜線。

修改後沒有更新

到儲存庫的 Actions 頁面查看部署是否完成，並確認最新提交已推送到 main。

完成任務：發布網站後，用手機網路開啟公開網址，逐一測試五個課程頁與複製按鈕，再把網址分享給朋友。
