Lesson 04・版本控制

Git 與 GitHub

Git 幫你保存每次修改；GitHub 則把專案存放在網路上，方便備份、分享與協作。

1. Git 和 GitHub 有什麼不同？

Git 是安裝在電腦上的版本控制工具，就像替專案建立可以回看的存檔點。GitHub 是存放 Git 專案的網站。

提交（commit）不是上傳。提交先記錄在自己的電腦；推送（push）才會把提交上傳到 GitHub。

2. 第一次使用 Git

安裝 Git 後，開啟終端機並設定顯示名稱與 Email。Email 建議與 GitHub 帳號相同。

git config --global user.name "你的名字"
git config --global user.email "you@example.com"
git config --global init.defaultBranch main

接著進入網站資料夾並建立 Git 儲存庫：

cd 網頁資料夾
git init

3. 建立第一個版本

status 查看改動，add 選擇要納入的檔案，commit 建立存檔點。

git status
git add .
git commit -m "建立初學者教學網站"

提交訊息要簡短描述「這次改了什麼」，例如「新增 Git 教學頁」，不要只寫 update。

4. 上傳到 GitHub

  1. 登入 GitHub，選擇 New repository。
  2. 輸入專案名稱，例如 frontend-starter。
  3. 先不要加入 README，建立空白儲存庫。
  4. 複製 GitHub 顯示的儲存庫網址，再執行以下指令。
git remote add origin https://github.com/你的帳號/frontend-starter.git
git branch -M main
git push -u origin main

第一次推送可能會要求登入 GitHub。完成後重新整理儲存庫頁面，就能看到網站檔案。

5. 之後的日常流程

每次完成一小段有意義的修改，就執行：

git status
git add .
git commit -m "說明這次的修改"
git push
練習：修改首頁的一段文字，查看 git status，建立提交並推送到 GitHub。到 GitHub 確認提交紀錄是否出現。
